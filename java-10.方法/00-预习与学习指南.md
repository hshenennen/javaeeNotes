# ✅  [[01- 方法的定义]]

### 🔹 什么是方法？

> **方法（Method）是完成特定功能的一段可重用代码块。**

它相当于“函数”，但 Java 中叫“方法”。

### 🔹 基本语法

```java
[修饰符] 返回值类型 方法名(参数列表) {
    // 方法体
    return 返回值; // 可选，取决于返回值类型
}
```

### 🔹 示例：计算两个数的和

```java
public static int add(int a, int b) {
    return a + b;
}
```

### 🔹 关键点解析

| 组成部分 | 说明 |
|--------|------|
| `public` | 访问修饰符，表示该方法可以被任何类访问 |
| `static` | 静态方法，属于类本身，不需要创建对象即可调用 |
| `int` | 返回值类型，表示这个方法返回一个整数 |
| `add` | 方法名，自定义，见名知意 |
| `(int a, int b)` | 参数列表，接收外部传入的数据 |
| `{...}` | 方法体，执行逻辑的地方 |

### 🔹 如何调用？

```java
int result = add(3, 5); // 调用方法，传参并接收返回值
System.out.println(result); // 输出: 8
```

---

### ✅ 学习建议

- 先从简单例子开始，比如打印信息、加减乘除。
- 注意区分：**有返回值 vs 无返回值**（`void`）。
- 练习：写一个方法输出“Hello World”；再写一个方法判断奇偶数。

# ✅ [[02- 方法的分类]]

### 🔹 按是否有返回值分类

| 类型 | 是否有返回值 | 示例 |
|------|--------------|------|
| `void` 方法 | 无返回值 | `printInfo()` |
| 有返回值方法 | 有返回值 | `getAge()`, `calculate()` |

### 🔹 按是否使用 `static` 分类

| 类型 | 是否需要实例化？ | 示例 |
|------|------------------|------|
| 静态方法（`static`） | 不需要 | `Math.max(a, b)` |
| 非静态方法（普通方法） | 需要先创建对象 | `student.getName()` |

> ⚠️ 后面会详细讲！

### 🔹 按功能分类

- 构造方法（Constructor）：用于创建对象
- 普通方法：实现业务逻辑
- getter/setter：获取/设置属性
- 主方法（`main`）：程序入口

- 画一张表格对比不同类型的差异。
- 尝试自己定义几个不同类型的方法，并测试调用方式。

# ✅ [[03-访问修饰符]]

### 🔹 四种访问级别（从严格到宽松）

| 修饰符 | 当前类 | 同包 | 子类 | 其他包 |
|--------|--------|------|------|--------|
| `private` | ✅ | ❌ | ❌ | ❌ |
| （默认） | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ |

### 🔹 实际应用示例

```java
public class Student {
    private String name;        // 私有字段，只能在本类访问
    public int age;             // 公共字段，任意地方都能访问
    
    public void study() {       // 公共方法，外部可调用
        System.out.println(name + " is studying");
    }
    
    private void sleep() {      // 私有方法，仅内部使用
        System.out.println("Sleeping...");
    }
}
```

### 🔹 常见规则

- 字段尽量设为 `private`，通过 `getter/setter` 控制访问。
- 方法根据需要选择合适的修饰符。
- `public` 一般用于对外接口；`private` 用于内部逻辑。

---

### ✅ 学习建议

- 用 UML 图或表格记忆四种修饰符的可见性。
- 写一个小类，尝试给每个成员加上不同的修饰符，观察编译错误。

# ✅ [[04-静态方法 vs 非静态方法]]

### 🔹 核心区别

| 特性 | 静态方法（`static`） | 非静态方法（普通方法） |
|------|-----------------------|--------------------------|
| 是否需要对象？ | ❌ 不需要 | ✅ 需要先创建对象 |
| 属于谁？ | 类 | 对象（实例） |
| 能否访问非静态成员？ | ❌ 不可以直接访问 | ✅ 可以 |
| 内存位置 | 类加载时分配 | 每个对象独立存储 |

### 🔹 示例对比

```java
public class MathUtils {
    public static int add(int a, int b) {
        return a + b; // ✅ 可以直接调用
    }
    
    public int subtract(int a, int b) {
        return a - b; // ❌ 不能直接访问 this.name
    }
}

// 调用方式
MathUtils.add(3, 5);           // ✅ 直接调用静态方法
MathUtils mu = new MathUtils(); 
mu.subtract(10, 3);            // ✅ 必须先创建对象
```

### 🔹 什么时候用静态方法？

- 工具类方法（如 `Math.max()`、`Arrays.sort()`）
- 不依赖对象状态的操作
- 作为程序入口（`main` 方法必须是 `static`）

### 🔹 什么时候用非静态方法？

- 操作对象的属性（如 `student.setName()`）
- 需要访问实例变量或方法
- 表达对象的行为

---

### ✅ 学习建议

- 动手写一个工具类（如 `Calculator`），包含静态和非静态方法。
- 尝试在静态方法中访问非静态变量 → 观察编译错误。
- 思考：为什么 `main` 方法是 `static`？

# ✅ [[05-方法重载]]

### 🔹 什么是方法重载？

> **在一个类中，允许存在多个同名方法，只要它们的参数列表不同（类型、数量、顺序）。**

### 🔹 重载规则

✅ 可以重载的条件：
- 方法名相同
- 参数列表不同（至少有一个不同）：
  - 参数个数不同
  - 参数类型不同
  - 参数顺序不同（不推荐）

❌ 不能重载的情况：
- 返回值类型不同（不行）
- 仅靠访问修饰符或 `static` 区分（不行）

### 🔹 示例

```java
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
       ,return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }

    public int add(int a, double b) { // 参数顺序不同也可以
        return (int)(a + b);
    }
}
```

### 🔹 调用时如何匹配？

Java 编译器会根据**实际传入的参数类型和数量**自动选择对应的方法。

```java
Calculator calc = new Calculator();
calc.add(1, 2);           // 调用 int + int
calc.add(1.5, 2.5);       // 调用 double + double
calc.add(1, 2, 3);        // 调用三个 int
```

---

### ✅ 学习建议

- 多写几个重载方法，练习编译器如何选择。
- 尝试让两个方法只差返回值类型 → 观察编译错误。
- 理解“重载”是为了提高代码的灵活性和可读性。
