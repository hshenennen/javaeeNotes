# 静态方法 vs 非静态方法

> **标签**：`#Java/方法` `#Java/static` `#Java/面向对象`

在 Java 中，方法根据是否使用 `static` 关键字，分为 **静态方法** 和 **非静态方法（实例方法）**。它们在调用方式、内存分配、访问权限等方面有本质区别。

---

## 核心区别对比表

| 特性 | 静态方法（Static Method） | 非静态方法（Instance Method） |
|------|--------------------------|------------------------------|
| **关键字** | 使用 `static` 修饰 | 无 `static` 修饰 |
| **所属对象** | 属于**类本身** | 属于**类的实例（对象）** |
| **调用方式** | `类名.方法名()` 或 `方法名()`（同类中） | 必须先 `new` 对象，再 `对象.方法名()` |
| **内存分配时机** | 类加载时分配（早于对象创建） | 对象创建时分配 |
| **访问成员限制** | 只能直接访问 **静态成员**（静态变量/方法） | 可访问 **所有成员**（静态 + 非静态） |
| **`this` / `super`** | 不能使用 `this` 或 `super` | 可以使用 `this` 和 `super` |
| **典型用途** | 工具方法（如 `Math.max()`）、辅助函数 | 操作对象状态（如 `person.getName()`） |

---

## 代码示例

```java
public class Student {
    private String name;               // 实例变量
    private static String school = "ABC High"; // 静态变量

    // 非静态方法（实例方法）
    public void setName(String name) {
        this.name = name; // ✅ 可访问实例变量
        System.out.println("School: " + school); // ✅ 也可访问静态变量
    }

    public String getName() {
        return this.name;
    }

    // 静态方法
    public static void printSchool() {
        System.out.println("School: " + school); // ✅ 可访问静态变量
        // System.out.println(name); // ❌ 编译错误！不能访问实例变量
        // this.name = "Tom";        // ❌ 不能使用 this
    }

    // main 方法（静态）——程序入口
    public static void main(String[] args) {
        // 调用静态方法：直接调用
        printSchool();

        // 调用非静态方法：必须先创建对象
        Student stu = new Student();
        stu.setName("Alice");
        System.out.println(stu.getName());
    }
}
```

## 常见问题解答

### ❓ 为什么 `main` 方法必须是 `static`？
- JVM 在启动程序时**不会创建类的实例**，而是直接调用 `main` 方法。
- 如果 `main` 不是静态的，JVM 就无法在没有对象的情况下调用它。

### ❓ 静态方法能调用非静态方法吗？
- **不能直接调用**。
- 但可以通过**创建对象**间接调用：

```java
public static void staticMethod() {
    Student s = new Student(); // 创建对象
    s.setName("Bob");          // 通过对象调用实例方法
}
```

### ❓ 非静态方法能调用静态方法吗？
- ✅ **可以**，直接调用即可（静态成员属于类，实例方法可访问类成员）。

```java
public void instanceMethod() {
    printSchool(); // ✅ 直接调用静态方法
}
```

## 设计建议 ✅

| 场景 | 推荐使用 |
|------|--------|
| 与对象状态无关的工具函数（如数学计算、字符串处理） | `static` 方法 |
| 需要操作对象属性或依赖实例状态的方法 | 非静态方法 |
| 工具类（如 `Math`、`Arrays`） | 全部使用 `static` 方法，并将构造器设为 `private` |

> 🛑 **避免滥用 static**：过度使用静态方法会降低代码的可测试性和灵活性（难以 mock，违反面向对象原则）。
