## ✅ 各章节预习指南

### 🔹 [[01-if 语句]]

> [!tip] 学习目标
> - 掌握 `if`、`if-else`、`if-else if-else` 的语法
> - 理解布尔表达式的作用
> - 能根据条件执行不同代码

#### 核心知识点：
```java
// 单分支
if (条件) {
    // 条件为 true 时执行
}

// 双分支
if (条件) {
    // true 时执行
} else {
    // false 时执行
}

// 多分支
if (条件1) {
    ...
} else if (条件2) {
    ...
} else {
    ...
}
```

#### 常见问题：
- 忘记大括号 `{}` → 只执行第一行
- `=` 与 `==` 混淆 → `if (a = 5)` 是赋值，不是判断
- 多层嵌套过深 → 逻辑复杂难读

#### 推荐练习：
```java
int score = 85;
if (score >= 90) {
    System.out.println("优秀");
} else if (score >= 80) {
    System.out.println("良好");
} else {
    System.out.println("需努力");
}
```

### 🔹[[02-switch语句]]

> [!tip] 学习目标
> - 掌握 `switch` 的基本用法
> - 理解 `case`、`break`、`default` 的作用
> - 了解 `switch` 与 `if` 的适用场景区别

#### 核心知识点：
```java
switch (变量) {
    case 值1:
        // 执行代码
        break;
    case 值2:
        // 执行代码
        break;
    default:
        // 默认情况
}
```

#### 特点：
- 仅支持 `byte`, `short`, `int`, `char`, `String`, `enum`
- `break` 防止穿透（fall-through）
- `default` 可选，建议保留

#### 常见问题：
- 忘记 `break` → 导致多个 `case` 执行
- `switch` 不支持浮点数或区间判断

#### 推荐练习：
```java
int day = 3;
switch (day) {
    case 1: System.out.println("周一"); break;
    case 2: System.out.println("周二"); break;
    case 3: System.out.println("周三"); break;
    default: System.out.println("其他");
}
```

### 🔹 [[03-for 循环]]

> [!tip] 学习目标
> - 掌握 `for` 循环的基本语法
> - 理解初始化、条件、更新三部分的作用
> - 能用于遍历数组、固定次数任务

#### 核心知识点：
```java
for (初始化; 条件; 更新) {
    // 循环体
}
```

#### 示例：
```java
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}
// 输出：0 1 2 3 4
```

#### 常见问题：
- 忘记初始化或更新 → 死循环
- 索引越界（如 `i <= arr.length`）

#### 推荐练习：
打印 1 到 10 的奇数。

### 🔹[[04-while循环]]

> [!tip] 学习目标
> - 掌握 `while` 循环的语法
> - 理解“先判断后执行”的特点
> - 能处理未知次数的循环（如用户输入）

#### 核心知识点：
```java
while (条件) {
    // 循环体
}
```

#### 特点：
- 先判断条件，再执行循环体
- 如果初始条件为 `false`，循环体**一次都不执行**

#### 常见问题：
- 忘记更新条件变量 → 死循环
- 初始化变量未定义 → 编译错误

#### 推荐练习：
让用户输入一个正整数，直到输入正确为止。

### 🔹[[ 05-do-while 循环]]

> [!tip] 学习目标
> - 掌握 `do-while` 的语法
> - 理解“先执行后判断”的特点
> - 能在必须至少执行一次的场景中使用

#### 核心知识点：
```java
do {
    // 循环体
} while (条件); // ⚠️ 注意结尾有分号！
```

#### 特点：
- **至少执行一次**，无论条件是否成立
- 适用于输入验证、菜单系统等场景

#### 常见问题：
- 忘记结尾分号 → 编译错误
- 无限循环风险（忘记更新条件）

#### 推荐练习：
实现一个简单的菜单，用户输入选项，直到选择退出。

### 🔹 [[06-break 和 continue 语句]]

> [!tip] 学习目标
> - 掌握 `break` 和 `continue` 的作用
> - 能在循环中合理使用它们
> - 理解标签（label）的使用（进阶）

#### 核心知识点：
| 语句 | 作用 |
|------|------|
| `break` | **立即退出当前循环或 switch** |
| `continue` | **跳过本次循环剩余代码，进入下一次** |

#### 示例：
```java
for (int i = 1; i <= 5; i++) {
    if (i == 3) {
        continue; // 跳过 i=3
    }
    System.out.print(i + " ");
}
// 输出：1 2 4 5
```

#### 常见问题：
- 在 `if` 中滥用 `break` → 编译错误
- `continue` 误以为会退出循环 → 实际只是跳过本次

#### 推荐练习：
编写程序，找出第一个大于 10 的偶数。

### 🔹 [[07-控制流程语句对比总结]]

> [!tip] 学习目标
> - 对比 `if` vs `switch`、`for` vs `while` vs `do-while`
> - 掌握每种语句的最佳使用场景
> - 能根据实际需求选择合适结构

#### 对比表格：

| 类型 | 何时使用 | 特点 |
|------|--------|------|
| `if` | 复杂条件、区间判断 | 灵活，支持任意布尔表达式 |
| `switch` | 离散值匹配（如菜单、状态码） | 性能高，结构清晰 |
| `for` | 已知循环次数 | 结构紧凑，适合遍历 |
| `while` | 未知次数，条件驱动 | 先判断，可能不执行 |
| `do-while` | 必须至少执行一次 | 先执行，再判断 |

#### 推荐练习：
给定以下需求，选择合适的语句：
- 用户输入密码，最多尝试 3 次 → `while` 或 `for`
- 菜单选择 → `switch`
- 遍历数组 → `for`
