## 📌 有参构造 vs 无参构造

### ✅ 基本定义

| 类型 | 定义 | 用途 |
|------|------|------|
| **无参构造方法** | 不接受任何参数的构造方法 | 用于创建对象时使用默认初始值 |
| **有参构造方法** | 接受一个或多个参数的构造方法 | 用于在创建对象时**直接传入初始数据** |

> 💡 构造方法名必须与**类名完全相同**，且**没有返回类型**。

---

### 💻 示例代码

```java
public class Student {
    private String name;
    private int age;

    // 🔹 无参构造方法
    public Student() {
        this.name = "未知";
        this.age = 0;
        System.out.println("调用了无参构造方法");
    }

    // 🔸 有参构造方法
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
        System.out.println("调用了有参构造方法");
    }

    public void introduce() {
        System.out.println("我是 " + name + "，今年 " + age + " 岁。");
    }
}
```

```java
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student();               // 调用无参构造
        Student s2 = new Student("小明", 18);     // 调用有参构造

        s1.introduce(); // 输出：我是 未知，今年 0 岁。
        s2.introduce(); // 输出：我是 小明，今年 18 岁。
    }
}
```

---

### ⚠️ 关键规则（必须掌握！）

#### 1. **默认构造方法的存在条件**
- 如果你**没有定义任何构造方法**，Java 编译器会**自动提供一个无参的默认构造方法**。
- 但只要你**定义了任意一个构造方法**（无论有参还是无参），**编译器就不再提供默认无参构造**！

✅ 示例：  
```java
class Book {
    // 没有任何构造方法 → 编译器自动加：public Book() {}
}
```

❌ 危险示例：
```java
class Book {
    public Book(String title) { ... } // 定义了有参构造
}
// 此时：new Book(); 会编译错误！因为无参构造不存在了！
```

> 🔸 **建议**：如果类需要无参构造（如用于框架、反射、序列化等），即使写了有参构造，也**显式写出无参构造**。

#### 2. **构造方法可以重载**
- 一个类可以有多个构造方法，只要**参数列表不同**（数量、类型、顺序）。
- 这称为 **构造方法重载（Constructor Overloading）**。

```java
public Student() { ... }
public Student(String name) { ... }
public Student(String name, int age) { ... }
```

#### 3. **使用 `this(...)` 避免重复代码**
在有参构造中调用无参构造（或反之），减少初始化逻辑重复：

```java
public Student() {
    this("匿名", 0); // 调用有参构造
}

public Student(String name, int age) {
    this.name = name;
    this.age = age;
}
```

> ⚠️ 注意：`this(...)` 必须是构造方法中的**第一条语句**。

### 🔁 对比总结表

| 特性 | 无参构造方法 | 有参构造方法 |
|------|--------------|--------------|
| 参数 | 无 | 有（一个或多个） |
| 主要用途 | 创建默认状态对象 | 创建带初始值的对象 |
| 是否自动生成 | 是（仅当无任何构造时） | 否（必须手动编写） |
| 是否可重载 | 可与其他构造共存 | 可与其他构造共存 |
| 常见场景 | 框架、反射、简单测试 | 实际业务中更常用 |
