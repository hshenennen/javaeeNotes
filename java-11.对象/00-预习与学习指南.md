### 📘 [[01-对象的定义]]
#### ✅ 预习问题
- 什么是“对象”？它和“变量”有什么区别？
- 如何在 Java 中创建一个对象？
- “实例”是什么意思？

#### 🧠 核心知识点
- **对象是类的实例**（Instance）
- 通过 `new` 关键字创建对象
- 对象包含状态（字段）和行为（方法）

```java
// 定义类
class Student {
    String name;
    int age;
}

// 创建对象
Student s = new Student();
s.name = "小明";
s.age = 18;
```

#### 💡 学习建议
- 画图表示：`类 → 实例化 → 对象`
- 练习：创建多个不同类型的对象（如 `Car`, `Book`）

---

### 📘 [[02- 类（Class）]]

#### ✅ 预习问题
- 类和对象的关系是什么？
- 类中可以有哪些成员？
- 如何定义一个完整的类？

#### 🧠 核心知识点
- **类是模板**，对象是具体实例
- 成员包括：
  - 字段（Field / Attribute）
  - 方法（Method / Behavior）
  - 构造器（Constructor）

```java
public class Student {
    // 字段
    private String name;
    private int age;

    // 方法
    public void study() {
        System.out.println(name + " 在学习");
    }

    // 构造器
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

#### 💡 学习建议
- 写一个自己的类（如 `Phone`, `Pet`）
- 尝试不写构造器，观察默认行为

### 📘 [[03-构造方法]]

#### ✅ 预习问题
- 什么是构造方法？它和普通方法有什么区别？
- 为什么需要构造方法？
- 构造方法的名称必须和类名一样吗？

#### 🧠 核心知识点
- **构造方法用于初始化对象**
- 无返回值，名称与类名相同
- 如果没有显式定义，编译器提供默认无参构造器

```java
public class Student {
    private String name;

    // 无参构造器
    public Student() {
        this.name = "未知";
    }

    // 有参构造器
    public Student(String name) {
        this.name = name;
    }
}
```

#### 💡 学习建议
- 尝试去掉所有构造器，看是否能创建对象
- 使用 `new Student()` 和 `new Student("张三")` 测试

### 📘[[04-有参构造 vs 无参构造]]

#### ✅ 预习问题
- 什么时候用有参构造？什么时候用无参？
- 能否同时存在多个构造器？
- 如何选择合适的构造器？

#### 🧠 核心知识点
| 类型 | 特点 | 适用场景 |
|------|------|----------|
| **无参构造** | 不传参数 | 测试、默认值 |
| **有参构造** | 传入初始值 | 实际业务中常用 |

```java
class Person {
    private String name;
    private int age;

    public Person() {
        this.name = "匿名";
        this.age = 0;
    }

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

#### 💡 学习建议
- 创建两个对象：`Person p1 = new Person();` 和 `Person p2 = new Person("李四", 20);`
- 观察输出结果差异

### 📘 [[05-static关键字]]

#### ✅ 预习问题
- `static` 是什么？它和普通成员有什么不同？
- 什么时候该用 `static`？
- `static` 成员属于谁？

#### 🧠 核心知识点
- **`static` 表示“属于类”而非“属于对象”**
- 静态成员在类加载时初始化，只有一份
- 可以通过类名直接访问：`ClassName.staticField`

```java
class Counter {
    private static int count = 0; // 静态字段
    private String name;

    public Counter(String name) {
        this.name = name;
        count++; // 每创建一个对象，计数器+1
    }

    public static int getCount() { // 静态方法
        return count;
    }
}
```

> ✅ 使用：
```java
Counter c1 = new Counter("A");
Counter c2 = new Counter("B");
System.out.println(Counter.getCount()); // 输出：2
```

#### 💡 学习建议
- 写一个 `MathUtil` 类，包含 `PI` 和 `add()` 静态方法
- 尝试在静态方法中访问非静态字段 → 编译错误！

### 📘 [[06-this 关键字]]

#### ✅ 预习问题
- `this` 是什么？它指向谁？
- 什么时候需要使用 `this`？
- `this` 和 `super` 有什么区别？

#### 🧠 核心知识点
- **`this` 指向当前对象实例**
- 三种常见用法：
  1. 区分局部变量和成员变量
  2. 调用本类其他构造器（`this(...)`）
  3. 返回当前对象（`return this;`）

```java
class Student {
    private String name;

    public Student(String name) {
        this.name = name; // 区分参数和字段
    }

    public Student setName(String name) {
        this.name = name;
        return this; // 支持链式调用
    }
}
```

> ✅ 使用：
```java
Student s = new Student("小明");
s.setName("小红").setName("小刚"); // 链式调用
```

#### 💡 学习建议
- 写一个带 `this` 的构造器，观察字段赋值过程
- 尝试省略 `this`，看是否能编译（可能冲突）

### 📘 [[07-this 的位置]]

#### ✅ 预习问题
- `this()` 可以放在构造器任意位置吗？
- `this()` 和 `super()` 可以共存吗？
- 构造器链是如何工作的？

#### 🧠 核心知识点
| 规则 | 说明 |
|------|------|
| **`this(...)` 必须是第一行** | 否则编译错误 |
| **不能与 `super(...)` 共存** | 二者只能选其一 |
| **构造器链最终必须调用 `super()`** | 或者父类有无参构造 |

```java
class Animal {
    public Animal() { System.out.println("Animal"); }
}

class Dog extends Animal {
    public Dog() {
        this("旺财"); // 调用兄弟构造器
    }

    public Dog(String name) {
        super(); // 调用父类构造器
        System.out.println("Dog: " + name);
    }
}
```

> ✅ 执行 `new Dog()` 输出：
```
Animal
Dog: 旺财
```

#### 💡 学习建议
- 尝试把 `this(...)` 放在第二行 → 编译错误
- 写一个三层继承结构，观察初始化顺序
