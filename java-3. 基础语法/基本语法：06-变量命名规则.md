# Java 变量命名规则

> [!info] 学习目标
> - 掌握 Java 变量命名的**强制规则**
> - 了解行业推荐的**命名规范（约定）**
> - 避免常见命名错误
> - 学会为不同类型的变量选择合适的名称

---

## 1. 强制规则（必须遵守）

Java 对变量名（标识符）有以下**语法要求**，违反会导致**编译错误**：

✅ **合法的变量名必须满足**：
1. **只能包含**：
   - 字母（A–Z, a–z）
   - 数字（0–9）
   - 下划线 `_`
   - 美元符号 `$`
2. **不能以数字开头**  
   ❌ `1name` → ✅ `name1`
3. **不能是 Java 关键字或保留字**  
   ❌ `int class = 5;`（`class` 是关键字）  
   ❌ `boolean true = false;`（`true` 是字面量）
4. **区分大小写**  
   `age`、`Age`、`AGE` 是三个不同的变量

### ✅ 合法示例
```java
int age;
double total_price;
String $message;
boolean isReady;
char _symbol;
```

### ❌ 非法示例
```java
int 2ndPlace;     // 以数字开头 ❌
double my-price;  // 包含连字符 ❌
boolean break;    // break 是关键字 ❌
float class;      // class 是关键字 ❌
```

---

## 2. 推荐命名规范（约定俗成）

虽然 Java 不强制要求，但遵循以下规范能让你的代码更**清晰、专业、易维护**。

### 📌 通用规则：**驼峰命名法（camelCase）**
- **首单词小写**，后续每个单词**首字母大写**
- 不使用下划线或连字符（除了常量）

| 变量类型 | 命名示例 | 说明 |
|--------|--------|------|
| 普通变量 | `studentName`, `maxSpeed`, `isActive` | 动词/形容词+名词组合 |
| 布尔变量 | `isFinished`, `hasPermission`, `canEdit` | 通常以 `is`/`has`/`can` 开头 |
| 循环计数器 | `i`, `j`, `k` | 仅限简单循环中使用 |

### 📌 常量命名：**全大写 + 下划线**
- 用 `final` 修饰的不可变变量（常量）
- 所有字母大写，单词间用 `_` 分隔

```java
final double PI = 3.14159;
final int MAX_RETRY_COUNT = 5;
final String DEFAULT_USERNAME = "guest";
```

### 📌 包名、类名、方法名对比（扩展了解）

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名（package） | 全小写，用点分隔 | `com.example.myapp` |
| 类名（class） | 大驼峰（PascalCase） | `Student`, `BankAccount` |
| 方法名（method） | 小驼峰（camelCase） | `calculateTotal()`, `isValid()` |
| **变量名** | **小驼峰（camelCase）** | **`userName`, `totalCount`** |

---

## 3. 命名建议：让名字“会说话”

> 💡 **好名字 > 注释**

| 差的命名 | 好的命名 | 为什么 |
|--------|--------|------|
| `int d;` | `int daysUntilExpiration;` | 明确表达含义 |
| `boolean f;` | `boolean fileExists;` | 避免无意义缩写 |
| `String str;` | `String customerEmail;` | 避免泛泛的名称如 `data`, `value`, `temp`（除非临时使用） |

> ✅ 原则：**看到变量名，就知道它是什么、用来做什么**

---

## 4. 特殊符号说明

- **`$` 和 `_` 虽然合法，但一般不推荐在普通变量中使用**  
  - `$` 常被编译器或框架自动生成（如内部类）
  - `_` 开头的变量可读性较差，除非有特殊约定

> 🚫 避免：`_name`, `$count`（除非你明确知道用途）

---

## 5. 常见错误总结

> [!danger] 初学者高频错误
> - 使用中文命名：`int 年龄 = 18;` → 虽然能编译，但**强烈不推荐**（可移植性差、团队协作困难）
> - 混淆 `0`（数字零）和 `O`（字母 O）：`int O = 10;` → 容易误读
> - 过度缩写：`usrNme` → 应写为 `userName`
> - 使用关键字：`int public = 5;` → 编译失败！

---

## 6. 小测验（判断对错）

以下变量名是否合法？是否推荐？

1. `int myVariable;` → ✅ 合法且推荐  
2. `double 3dPoint;` → ❌ 非法（数字开头）  
3. `boolean is_valid;` → ⚠️ 合法，但**不推荐**（应使用驼峰：`isValid`）  
4. `String URL;` → ⚠️ 合法，但常量应全大写：`final String URL = "...";`  
5. `char $c;` → ✅ 合法，但不推荐（`$` 少用）
