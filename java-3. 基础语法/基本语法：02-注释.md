
# Java 注释

在 Java 中，注释（Comments）是用来解释代码、提高可读性、便于维护的非执行文本。Java 支持三种类型的注释。

---

## 1. 单行注释（Single-line Comment）

使用 `//` 开头，注释内容从 `//` 开始到该行结束。

```java
// 这是一个单行注释
int x = 10; // 这也是单行注释，跟在代码后面
```

---

## 2. 多行注释（Multi-line Comment）

使用 `/* ... */` 包裹，可以跨越多行。

```java
/*
这是一个多行注释。
可以写很多行，
不会被编译器执行。
*/
int y = 20;
```

> [!warning] 注意
> 多行注释不能嵌套，例如 `/* /* 错误 */ */` 会导致编译错误。

---

## 3. 文档注释（Javadoc）

使用 `/** ... */`，专门用于生成 API 文档。通常放在类、方法、字段等声明之前。

```java
/**
 * 这是一个用于计算两个整数之和的工具类。
 * @author YourName
 * @version 1.0
 */
public class Calculator {

    /**
     * 返回两个整数的和。
     * @param a 第一个整数
     * @param b 第二个整数
     * @return 两数之和
     */
    public int add(int a, int b) {
        return a + b;
    }
}
```

使用 `javadoc` 工具可自动生成 HTML 文档：

```bash
javadoc Calculator.java
```

---

### 常用 Javadoc 标签

| 标签 | 说明 |
|------|------|
| `@param` | 描述方法参数 |
| `@return` | 描述返回值 |
| `@throws` / `@exception` | 描述可能抛出的异常 |
| `@author` | 作者信息 |
| `@version` | 版本信息 |
| `@see` | 参考链接或其他类/方法 |
| `@deprecated` | 标记已过时的代码 |

---

## 最佳实践

- ✅ 对 `public` API 使用 Javadoc。
- ✅ 对复杂逻辑添加清晰的单行或多行注释。
- ❌ 避免“废话注释”，如 `i++; // i 自增`。
- 💡 注释应解释 **“为什么”**，而不是 **“做什么”**（代码本身应足够清晰表达“做什么”）。
